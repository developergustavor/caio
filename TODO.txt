[x] - git (svn)
  - configuração de credenciais para múltiplos usuários (.gitconfig)
  - .gitignore (colocar os nomes dos arquivos ou pastas que não quer fazer upload)
  - .gitkeep (colocar os nomes dos arquivos ou pastas que quer fazer upload)
  $ git init: inicializa um repositório LOCAL (offline) vazio na pasta onde rodou o comando (sempre rode na pasta raiz do projeto)
  $ git remote add origin https://developergustavor@github.com/developergustavor/caio.git : linka o repositório local com o repositório que está no github
  $ git add . (adiciona os arquivos no repositório local)
  $ git commit -m "*prefixo (fix, chore, feat, etc)*: *titulo*                                                   
    ➜ - *topico 1*
    ➜ - *topico 1*"
  $ git clone https://developergustavor@github.com/developergustavor/caio.git : clona o repositório para sua máquina local
  $ git branch --all : listagem dos branchs
  $ git fetch : syncar seu repositório local com o remoto
  $ git switch *branch_destino* : troca de branch
  $ git pull origin *nome_branch_atual* : atualiza o repositório local de acordo com o branch remoto

  - pull requests: abertura de um "ticket" de revisão de código/alterações visando o controle, estabilidade e organização dos branchs
  - merge requests: "ticket" de fusão de branchs. Executada automaticamente quando uma pull request é validada

  - branchs: 
    - master: branch estável de produção, não pode commitar nele direto
    - dev: branch estável de homologação, não pode commitar nele direto, somente ele comunica com o master
    - feat/login-page: branch que o objetivo é criar uma feature de uma tela de login (caio trabalhando nele), somente comunica com o dev
    - chore/home-page: branch que o objetivo é criar uma feature de uma tela home (gustavo trabalhando nele), somente comunica com o dev
